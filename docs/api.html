AuthRequest

AuthRequest represents the authentication request for both registration and login.

FieldTypeLabelDescription
email string

[REQUIRED] [MAX LEN 255] Email must be a valid email address format (e.g., "user@example.com"). Example: "john.doe@company.com"

password string

[REQUIRED] [MIN LEN 8] [MAX LEN 32] Password is hashed on the server before storing. Example: "MySecurePass123!"

AuthResponse

AuthResponse represents the server's response to successful authentication.

FieldTypeLabelDescription
user_id string

Unique identifier for the user. Format: UUID v4. Example: "123e4567-e89b-12d3-a456-426614174000"

email string

Email address associated with the authenticated user. Email must be a valid email address format (e.g., "user@example.com"). Example: "john.doe@company.com"

token string

JWT token for subsequent authenticated requests. Format: JWT string (header.payload.signature). Must be included in subsequent requests as "authorization" metadata. Valid for: 24 hours by default. Example: "eyJhbGciOiJIUzI1NiIs..."

VerifyTokenRequest

VerifyTokenRequest represents a token verification request.

FieldTypeLabelDescription
token string

[REQUIRED] JWT token to verify. Must be a valid JWT token previously issued by the auth service. Example: "eyJhbGciOiJIUzI1NiIs..."

VerifyTokenResponse

VerifyTokenResponse contains user information if token is valid.

FieldTypeLabelDescription
user_id string

User ID associated with the token. Format: UUID v4. Example: "123e4567-e89b-12d3-a456-426614174000"

email string

Email address associated with the user. Format: valid email address. Example: "john.doe@company.com"

AuthService

AuthService provides authentication and authorization functionality.

Method NameRequest TypeResponse TypeDescription
Register AuthRequest AuthResponse

Register creates a new user account. Errors: (INVALID_ARGUMENT): If email format is invalid or password doesn't meet requirements (ALREADY_EXISTS): If the email is already registered (INTERNAL): For server-side errors
Login AuthRequest AuthResponse

Login authenticates an existing user. Errors: (INVALID_ARGUMENT): If email format is invalid (NOT_FOUND): If the email is not registered (UNAUTHENTICATED): If the password is incorrect (INTERNAL): For server-side errors
VerifyToken VerifyTokenRequest VerifyTokenResponse

VerifyToken validates a JWT token and returns associated user information. Errors: (INVALID_ARGUMENT): If token format is invalid (UNAUTHENTICATED): If token is expired or invalid (INTERNAL): For server-side errors

ConnectionTypeRequest

ConnectionTypeRequest represents a request to create a connection type.

FieldTypeLabelDescription
id string

[OPTIONAL] [MAX LEN 255] [FORMAT UUID v4] Unique identifier for the connection type, recieved by the FindConnectionTypes endpoint. You cannot create your own ID. If provided for CREATE operations, server will link the users version of the connection type to the shared connection type. Required for UPDATE operations. Example: "123e4567-e89b-12d3-a456-426614174000"

name string

[REQUIRED] [MAX LEN 255] Name for the users version of this connection type. Can be seen by other users. Example: "Works at" or "Brother"

definition string

[REQUIRED] [MAX LEN 4096] Description for the users version of this connection type. Should provide clear, comprehensive information about the connection type, can be seen by other users. Example: "Represents a current employment relationship between a person and a company"

ConnectionTypesList

FieldTypeLabelDescription

List of connection types matching the search criteria. May be empty if no matches are found

Empty

Empty message for requests/responses that don't need any data

GraphService

GraphService provides operations for managing graph-based knowledge representation.

All operations require authentication via JWT token in the "authorization" metadata.

Method NameRequest TypeResponse TypeDescription
CreateUser UserRequest Empty

CreateUser initializes a new user in the graph service. This endpoint is only accessible by the auth service. Errors: (INVALID_ARGUMENT): If user_id format is invalid (ALREADY_EXISTS): If user already exists (INTERNAL): For server-side errors
GetUserData Empty UserData

GetUserData retrieves all entities, connection types, and property types associated with the authenticated user. Errors: (UNAUTHENTICATED): If authentication is missing or invalid (INTERNAL): For server-side errors
CreateEntity EntityRequest UsersEntity

CreateEntity creates a new entity or links to an existing one if ID is provided. Errors: (INVALID_ARGUMENT): If name or definition exceed length limits (NOT_FOUND): If entity ID is provided but entity doesn't exist (UNAUTHENTICATED): If authentication is missing or invalid (INTERNAL): For server-side errors
UpdateEntity EntityRequest Empty

UpdateEntity modifies an existing entity. Errors: (INVALID_ARGUMENT): If name or definition exceed length limits (NOT_FOUND): If entity doesn't exist (UNAUTHENTICATED): If authentication is missing or invalid (INTERNAL): For server-side errors
FindEntities SearchRequest EntitiesList

FindEntities searches for entities by exact name match. Errors: (INVALID_ARGUMENT): If name is empty or too long (UNAUTHENTICATED): If authentication is missing or invalid (INTERNAL): For server-side errors
CreateConnectionType ConnectionTypeRequest UsersConnectionType

CreateConnectionType creates a new connection type or links to an existing one. Errors: (INVALID_ARGUMENT): If name or definition exceed length limits (NOT_FOUND): If entity ID is provided but entity doesn't exist (UNAUTHENTICATED): If authentication is missing or invalid (INTERNAL): For server-side errors
FindConnectionTypes SearchRequest ConnectionTypesList

FindConnectionTypes searches for connection types by exact name match. Errors: (INVALID_ARGUMENT): If name is empty or too long (UNAUTHENTICATED): If authentication is missing or invalid (INTERNAL): For server-side errors
